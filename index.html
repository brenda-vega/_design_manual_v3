01 — Estrategia

Product Thinking

Responde esto antes de tocar Figma. Si no puedes contestarlo, no diseñes aún.
5 preguntas obligatorias
¿Quién es el usuario?
Perfil, contexto, nivel técnico
¿Qué problema REAL tiene?
No feature. Problema de fondo
¿Qué quiere lograr?
Job to be done
¿Qué lo frena hoy?
Fricción actual
Fórmula de decisión
Voy a cambiar [X]
para lograr [Y]
porque el usuario [Z]
Ejemplo: Voy a usar iconos filled en navegación / para indicar estado activo / porque el usuario necesita feedback claro de ubicación sin tener que leer el label.
Framework express
Problema→ Usuario→ Decisión→ UI→ Resultado esperado
Regla base
Ningún elemento visual existe sin función.
Icono, color, sombra, espacio — todo tiene razón de existir. Si no puedes defenderlo, quítalo.

05 — Sistema

Motion Tokens

El movimiento tiene propósito. No decora — informa.
Principio base
El motion debe comunicar: cambio de estado, jerarquía, origen/destino, feedback de acción.
Si una animación no comunica algo útil → quítala.
Tokens de duración
TokenmsCuándo usar
duration/instant0Cambios sin transición
duration/fast-0170Micro-feedback: ripple, highlight, hover de ícono
duration/fast-02110Hover state de botón, focus ring appearance
duration/moderate-01150Fade de tooltip, aparición de dropdown pequeño
duration/moderate-02240Expansión de accordion, slide de panel pequeño
duration/slow-01300Modal enter/exit, sidebar open, page transition
duration/slow-02400Sheet completo, drawer, overlay de pantalla

easing/standard-accelcubic-bezier(0.3, 0, 1, 1)Elementos que salen → aceleran al irse (modal exit).

Accesibilidad
WCAG 2.3.3 — prefers-reduced-motion
En código: @media (prefers-reduced-motion: reduce) → eliminar o reducir transiciones.

En Figma: documenta en el handoff qué animaciones deben desactivarse. Aplica a: parallax, auto-play, animaciones de bucle, transiciones de página agresivas.

07 — Calidad

WCAG 2.2 / EN 301 549

Estándar técnico EN 301 549, alineado con WCAG 2.2 Nivel AA. Reconocido como ISO/IEC 40500:2025.
Contraste — ratios mínimos
Tipo de elementoCriterioRatio AARatio AAA
Texto normal (<18px / <14px bold)1.4.34.5:17:1
Texto grande (≥18px / ≥14px bold)1.4.33:14.5:1
Componentes UI (inputs, botones, iconos)1.4.113:1—
Decorativo / sin información—Sin requisito—
💡
Verifica contraste en modo claro y oscuro por separado. Plugin recomendado: Contrast o Stark en Figma.
Touch target — WCAG 2.5.8
44 × 44 px
Si el elemento visual es más pequeño, amplía el área de toque con padding invisible.
Aplica a: botones, links, checkboxes, toggles, iconos interactivos.

Excepción: targets menores a 24×24px son aceptables si tienen 24px de espacio libre entre ellos.

Regla de error
Error = color rojo + ícono + texto descriptivo
WCAG 1.4.1: nunca dependas solo del color para comunicar un error. Un usuario con deuteranopia no lo detectará.
08 — Calidad

UI Principles & Heurísticas

Referencia rápida. Consulta antes de tomar decisiones de diseño.
7 principios UI
#PrincipioAplicación práctica
1JerarquíaTamaño, peso y contraste guían la atención. Lo importante va primero y más grande.
2Divulgación progresivaNo muestres todo a la vez. Revela información conforme el usuario avanza.
3ConsistenciaMismo componente → mismo comportamiento en toda la app.
4ContrasteUsa contraste alto para lo crítico, bajo para lo secundario. Nunca al revés.
5AccesibilidadWCAG 2.2 AA mínimo. Diseña para todos desde el inicio, no como parche final.
6ProximidadLo relacionado va junto. La distancia entre elementos comunica relación.
7AlineaciónGrid siempre. Sin elementos flotando sin anclaje a la cuadrícula.
10 heurísticas de Nielsen
#HeurísticaPregunta de control
1Visibilidad del estado¿El usuario sabe qué está pasando en todo momento?
2Match con el mundo real¿Usas lenguaje y metáforas que el usuario conoce?
3Control y libertad¿Puede deshacer/salir fácilmente?
4Consistencia¿Mismo elemento = misma función siempre?
5Prevención de errores¿El diseño evita que el error ocurra antes de corregirlo?
6Reconocer, no recordar¿El usuario ve las opciones o las tiene que memorizar?
7Flexibilidad¿Hay atajos para usuarios avanzados?
8Diseño minimalista¿Cada elemento tiene función? ¿Qué puedo quitar?
9Mensajes de error claros¿El error explica qué pasó y cómo solucionarlo?
10Ayuda contextual¿Existe ayuda cuando el usuario la necesita?

Checklist antes de exportar
ItemVerificar
ÍconosTamaño en múltiplo de 2 (16, 18, 20, 24, 32). Stroke outlined, no expandido.
ÍconosViewport exacto sin espacio extra. Fill none donde corresponde.
ImágenesExportar 1× y 2×. Convertir a WebP. Verificar peso final.
ImágenesAlt text documentado en la capa de Figma.
TokensSeparar primitivos de semánticos. Verificar referencias no rotas.
TokensExportar ambos sets: light.json y dark.json.
Namingkebab-case, sin espacios, sin caracteres especiales.
NamingVariante con double dash: nombre--variante. Densidad con @: nombre@2x
Herramientas recomendadas
Optimización SVG
SVGO — CLI para comprimir. SVGOMG — versión web. Reducir paths, eliminar metadata.
Conversión WebP
squoosh.app — drag & drop en browser. cwebp — CLI. ImageOptim — macOS.
Tokens
Tokens Studio (Figma plugin) — export JSON. Style Dictionary — transform a CSS/iOS/Android.
Contraste
Contrast (Figma plugin) — check en selección. Stark — simulación de daltonismo. coolcontrast.com
Inspect
Figma Dev Mode — specs automáticas. Zeplin — alternativa con anotaciones adicionales.
Lottie
LottieFiles — preview y optimización. lottie-web — implementación JS. BodyMovin — export desde AE.
